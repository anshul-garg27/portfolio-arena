Projects, Failures Included

Every project has a story. Most of those stories include at least one moment of "oh no."

PRJ_001 SHIPPED

E-Commerce Platform

A full-stack commerce engine with real-time inventory, Stripe integration, and an admin dashboard that doesn't make you want to throw your laptop. Built because I wanted to understand every layer of a transaction.

WHAT WENT WRONG: First version had a race condition in the cart service. Two users could "buy" the last item simultaneously. Discovered this during a demo. The silence was deafening.
WHAT I LEARNED: Optimistic locking, database transactions, and the importance of load testing before you show anyone anything. Also: always have a backup demo plan.
react node.js postgresql stripe redis
PRJ_002 SHIPPED

Real-Time Analytics Dashboard

WebSocket-powered data viz that actually updates without you refreshing the page like it's 2005. Custom chart components because D3 is powerful but also wants to hurt you.

WHAT WENT WRONG: Memory leak. The WebSocket connections weren't being cleaned up on component unmount. After 30 minutes, the browser tab was using 2GB of RAM. Chrome was not happy. Neither was I.
WHAT I LEARNED: useEffect cleanup functions aren't optional. Memory profiling in Chrome DevTools. The importance of disconnect handlers. Basically: clean up after yourself.
next.js d3.js websockets graphql
PRJ_003 EVOLVING

Auth Microservice

Because "just use Firebase" felt like cheating, and I wanted to truly understand OAuth flows, JWT refresh token rotation, and why security is so much harder than it looks.

WHAT WENT WRONG: V1 stored refresh tokens in localStorage. Yes, I know. I know now. XSS vulnerability waiting to happen. A security-minded friend reviewed it and the code review was... educational.
WHAT I LEARNED: HttpOnly cookies, token rotation, the OWASP Top 10, and humility. Rewrote the entire token management layer. Twice. It's actually solid now.
node.js redis docker jwt oauth2
